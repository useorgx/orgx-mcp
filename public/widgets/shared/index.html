How Memory Flows

Every conversation builds organizational memory. Work from one session automatically enriches the next—across tools and domains.

1

Work in Any Tool

Create a feature spec in Claude. Draft marketing copy in ChatGPT. Build code in Cursor.

2

Auto-Captured & Tagged

OrgX automatically saves artifacts and tags them by domain—Product, Engineering, Marketing, Sales.

3

Context Flows Forward

When Marketing drafts the launch, they pull from the same spec. No re-explaining needed.

Protocol Capabilities

Full organizational control across 5 domains. Read context. Take action.

Decisions

Read Write

View pending decisions, approve or reject with notes, and access decision history.

get_pending_decisions() approve_decision(id, note) reject_decision(id, reason)

Agents

Read Write

Monitor agent status, spawn new tasks, and configure agent behavior.

get_agent_status() spawn_agent_task(agent, task) configure_org(action, agent_type)

Initiatives

Read Write

Track initiative health, create new workstreams, and manage entity lifecycles.

get_initiative_pulse() create_entity(type, data) entity_action(type, id, action)

Memory

Read

Search across all artifacts, decisions, and conversations with semantic understanding.

query_org_memory(query) get_org_snapshot()

Planning

Read Write

Start collaborative planning sessions, iterate on plans, and finalize initiatives.

start_plan_session(goal) improve_plan(session_id) complete_plan(session_id)
